# OpenEMR Patient Registration Manual Testing Project

## Overview

This repository showcases an end-to-end Manual Testing project performed on the Patient Registration module of OpenEMR, an open-source Electronic Health Record (EHR) system used in the Healthcare domain.

The project follows a structured Software Testing Life Cycle (STLC) approach, covering test planning, test design, test execution, defect reporting, and documentation.

---

## Application Details

**Application:** OpenEMR

**Domain:** Healthcare

**Module Tested:** Patient Registration (Demographics)

**Application Type:** Electronic Health Record (EHR) System

**Demo Environment:** https://www.open-emr.org/demo/

---

## Project Workflow

### Step 1: Test Planning

Prepared a comprehensive Test Plan document defining:

* Scope of Testing
* Test Objectives
* Test Strategy
* Entry and Exit Criteria
* Risks and Assumptions
* Test Environment
* Deliverables

**Artifact:**
`01_Test_Plan_OpenEMR_Patient_Registration.docx`

---

### Step 2: Test Scenario & Test Case Design

Designed detailed test scenarios and test cases covering:

* Positive Test Scenarios
* Negative Test Scenarios
* Mandatory Field Validation
* Data Validation Rules
* Functional Workflows
* User Input Validation

**Artifact:**
`02_Test_Scenarios_and_Test_Cases_OpenEMR.xlsx`

---

### Step 3: Test Execution

Executed all designed test cases against the OpenEMR application and documented:

* Actual Results
* Expected Results
* Pass/Fail Status
* Execution Observations

**Artifact:**
`03_Test_Execution_Report_OpenEMR.xlsx`

---

### Step 4: Defect Reporting

Identified and documented defects discovered during testing, including:

* Defect Description
* Steps to Reproduce
* Expected Result
* Actual Result
* Severity
* Priority
* Defect Status

**Artifact:**
`04_Defect_Report_OpenEMR.xlsx`
